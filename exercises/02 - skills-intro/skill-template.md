# README Standard

## When to Use

- Generating a README for a new project
- Updating an existing README after significant changes
- Reviewing a README for completeness

## Required Sections (in order)

1. **Title and description** — project name as H1, one sentence explaining
   what it does and why it exists.
2. **Quick Start** — minimal steps to get running locally. Assume the reader
   knows Python.
3. **Configuration** — table of environment variables with defaults.
4. **API Reference** — for each endpoint: method, path, description, example
   request (curl), example response.
5. **Project Structure** — file tree with one-line descriptions of key files.
6. **Documentation** — link to specs or design docs if they exist in the repo.
7. **Running Tests** — how to run the test suite.
